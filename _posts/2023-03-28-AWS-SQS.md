# Application Integration



## AWS SQS 소개

* **SQS = Simple Queue Service**
* **완전 관리형 분산 메시지 큐 서비스**
  * 마이크로 서비스, 분산 시스템 및 서버리스 애플리케이션을 분리하고 확장할 수 있음
* 분산된 소프트웨어 구성 요소와 서비스 간에 메시지를 전송, 수신 및 처리할 수 있는 안정적이고 가용성이 높으며 확장 가능한 클라우드 기반 메시징 서비스를 제공합니다.
* AWS SQS는 클라우드에서 확장 가능하고 안정적인 애플리케이션을 구축할 수 있도록 지원하는 강력하고 유연한 메시징 서비스입니다.

## AWS SQS 특징

- 완전 관리형, 애플리케이션 분리하는 데 사용
- 무제한 처리량 -> 무한적으로 확장 가능
  - 초 당 원하는 만큼의 메시지를 전송할 수 있으며 대기열에도 원하는 만큼의 메시지를 포함시킬  수 있음
- 각 메시지는 4일(기본값) ~ 14일동안 대기열에 있을 수 있음
- latency 짧음 (10 ms 미만)
- 하나의 메세지 당 256KB 까지 가능하며 Queue에 최대 14일까지 남음
- 대기열 서비스이므로 중복 메시지 있을 수 있음
- best effort ordering

## AWS SQS의 이점

* 확장성
  * AWS SQS는 확장성이 뛰어나도록 설계
  * 용량 계획 또는 프로비저닝에 대한 걱정 없이 대량의 메시지와 트래픽을 처리 가능

* 내구성
  * AWS SQS는 지역 내 여러 가용성 영역(AZ)에서 메시지를 복제하여 단일 AZ가 다운되더라도 메시지가 손실되지 않도록 보장

* 메시지 보존
  * AWS SQS를 사용하면 메시지를 대기열에 유지할 시간을 지정할 수 있음(다양한 시간에 메시지를 처리 가능)

* 보안
  * AWS SQS는 유휴 및 전송 중인 데이터의 암호화, IAM 정책 및 SSL/TLS 암호화 지원을 비롯한 다양한 보안 기능을 제공

* Low Latency
  * AWS SQS는 Low Latency Messaging 기능을 제공하여 매우 짧은 지연 시간으로 메시지를 주고받을 수 있음
  * 실시간 애플리케이션에 적합

* 비용 효율적
  * AWS SQS는 초기 비용이나 장기 계약에 대한 걱정 없이 사용하는 리소스에 대해서만 비용을 지불할 수 있는 비용 효율적인 솔루션
