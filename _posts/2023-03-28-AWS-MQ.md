## AWS MQ 소개

* **MQ = Message Queue**
* 관리형 메시지 브로커 서비스입니다. 
* MQTT, AMQP 및 STOMP와 같은 널리 사용되는 메시징 프로토콜을 지원합니다.
* 메시지 기반 애플리케이션을 위한 가용성, 내구성 및 확장성이 뛰어난 인프라를 제공합니다.
* AWS MQ는 클라우드에서 확장 가능하고 안정적인 애플리케이션을 구축할 수 있도록 지원하는 강력하고 유연한 메시징 서비스입니다.
* AWS SQS, AWS SNS 만큼 스케일링은 하지 않습니다.

## AWS MQ의 주요 기능과 이점

* 프로토콜 지원
  * AWS MQ는 MQTT, AMQP, STOMP 등 다양한 메시징 프로토콜을 지원하므로 사용자의 요구에 가장 적합한 프로토콜을 선택할 수 있습니다.

* 관리형 서비스

  * AWS MQ는 확장성, 고가용성 및 내구성을 비롯한 기본 인프라를 관리하는 완벽한 관리형 서비스이므로 애플리케이션 개발에 집중할 수 있습니다.

  > **고가용성(High Availablity)**
  >
  > 각각의 가용영역(AZ)에 active, standby 인스턴스 존재 (이중화 구성)

* AWS 서비스와의 통합
  * AWS MQ는 Amazon SNS, Amazon SQS, AWS Lambda 등 다른 AWS 서비스와 통합되어 여러 AWS 서비스를 활용하는 복잡한 애플리케이션을 쉽게 구축할 수 있습니다.

* 보안
  * AWS MQ는 저장 및 전송 중인 데이터의 암호화, IAM 정책 및 SSL/TLS 암호화 지원을 비롯한 다양한 보안 기능을 제공합니다.

* 유연성
  * AWS MQ는 유연하게 설계되었으며 간단한 포인트 투 포인트 메시징에서 복잡한 게시-구독 아키텍처에 이르기까지 다양한 사용 사례에 사용할 수 있습니다.

* 비용 효율적
  *  AWS MQ는 초기 비용이나 장기 계약에 대한 걱정 없이 사용하는 리소스에 대해서만 비용을 지불할 수 있는 비용 효율적인 솔루션입니다.
