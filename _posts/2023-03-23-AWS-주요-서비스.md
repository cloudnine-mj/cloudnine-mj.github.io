## :fire: AWS Cloud 주요 서비스

* **Amazon EC2**: 클라우드의 가상 서버

* **Amazon Simple Storage Service (S3)**:  클라우드에서의 확장 가능한 스토리지

* **Amazon Aurora**: MySQL 및 PostgreSQL과 완벽하게 호환되는 고성능 관리형 관계형 데이터베이스

* **Amazon DynamoDB**: 관리형 NoSQL 데이터베이스

* **Amazon RDS**: MySQL, PostgreSQL, Oracle, SQL Server 및 MariaDB를 위한 관리형의 관계형 데이터베이스 서비스

* **AWS Lambda**: 서버에 대한 걱정없이 코드 실행

* **Amazon Lightsail**: 가상 프라이빗 서버 시작 및 관리

* **Amazon SageMaker**: 기계 학습 모델을 대규모로 구축, 훈련 및 배포



## :mag_right: AWS로 시작하는 서비스 VS Amazon으로 시작하는 서비스

* 이 부분에 대해서는 AWS에서도 공식적인 답변을 낸 적은 없음
* AWS 강의실 ([https://www.youtube.com/@AWSClassroom](https://www.youtube.com/@AWSClassroom)) 강의 영상 보고 참고용으로 정리해보았음.

### Amazon xxx VS AWS xxx

<img src = "https://github.com/cloudnine-mj/Titanic-survival-prediction/assets/113915835/f9c5b5be-9783-425d-bac5-bcaf5eb1b7f5" width = "80%">

* Amazon으로 시작하는 서비스는 AWS의 기본이 되는 Primitive 서비스
  * 자동차로 따지면 엔진, 타이어, 브레이크 등 가장 기초적인 구성요소가 되는 서비스들
  * Amazon EC2, Amazon S3, Amazon SQS, Amazon SNS, Amazon API Gateway 등
* AWS로 시작하는 서비스는 다양한 AWS 서비스로 구성해서 구체적인 기능을 제공하는 Wrapping 서비스
  * 엔진, 바퀴, 기어 등으로 만든 특수목적의 자동차 / 중장비 같은 것들
  * 바퀴, 엔진 특수 장비들을 잘 섞어서 아스팔트 바닥을 평평하게 만드는 목적을 가진 로드롤러를 만드는 것
* 하지만 AWS Lambda, AWS IAM처럼 AWS로 시작해도 기초서비스인 경우도 있고,  Amazon Athena, Amazon  RDS, Amazon Cognito 처럼 여러 서비스로 구성되어 있는데 Amazon으로 시작하는 경우도 있음.
