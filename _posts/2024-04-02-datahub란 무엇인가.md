# 1. DataHub란 무엇인가?

## 개념

* DataHub는 LinkedIn에서 개발한 오픈소스 메타데이터 플랫폼으로, 조직의 모든 데이터 자산을 중앙에서 관리하고 검색할 수 있게 해주는 데이터 카탈로그 시스템
* 데이터의 계보(lineage), 스키마, 소유자, 태그 등의 메타데이터를 자동으로 수집하고 시각화하여 데이터 거버넌스와 데이터 발견성을 향상시킴.

## 핵심 기능

**1. 메타데이터 자동 수집**

- 다양한 데이터 소스(MySQL, PostgreSQL, Kafka, Airflow, Snowflake, BigQuery 등)에서 자동으로 메타데이터 수집
- 스키마 정보, 테이블 통계, 사용 패턴 등을 주기적으로 업데이트

**2. 데이터 계보(Lineage) 추적**

- 데이터가 어디서 생성되어 어떻게 변환되고 어디로 흘러가는지 시각화
- 업스트림/다운스트림 의존성을 그래프로 표현
- 영향도 분석 (Impact Analysis) 가능

**3. 통합 검색**

- 모든 데이터 자산을 하나의 검색 인터페이스에서 조회
- Elasticsearch 기반의 강력한 풀텍스트 검색
- 태그, 소유자, 도메인 등으로 필터링

**4. 데이터 거버넌스**

- 데이터 소유자 지정 및 관리
- 태그 기반 분류 (PII, 민감정보 등)
- Glossary로 비즈니스 용어 정의

**5. 협업 기능**

- 데이터셋에 대한 문서화 및 설명 추가
- 데이터 품질 이슈 및 논의 기록
- 데이터 자산에 대한 피드백 공유

## 주요 구성 요소

**1. GMS (Generalized Metadata Service)**

- 메타데이터 저장 및 조회를 담당하는 핵심 서비스
- REST API 제공
- Spring Boot 기반

**2. Frontend**

- 사용자 인터페이스 서버
- Play Framework 기반
- GraphQL API 제공

**3. Metadata Store**

- MySQL 또는 PostgreSQL에 메타데이터 저장
- Entity-Aspect 모델로 구조화

**4. Search Index**

- Elasticsearch에 검색 인덱스 저장
- 빠른 검색 및 필터링 지원

**5. Message Queue**

- Kafka로 메타데이터 변경 이벤트 전송
- 비동기 처리 및 이벤트 기반 자동화

**6. Ingestion Framework**

- 다양한 소스에서 메타데이터 수집
- Python 기반 확장 가능한 구조

### 활용 예시

**1. 데이터 검색 및 발견**

```
데이터 팀 (예시)		 	→ "메트릭 데이터는 어디...?" 
              → DataHub 검색: "metadata"
              → sales.customer_orders 테이블 발견
              → 소유자, 스키마, 샘플 데이터 확인
```

**2. 영향도 분석**

```
엔지니어 → users 테이블 스키마 변경 예정
        → DataHub Lineage 확인
        → 다운스트림: 15개 테이블, 7개 대시보드 영향
        → 관련 팀에 사전 공지
```

**3. 데이터 거버넌스**

```
보안팀 → PII 데이터 식별 필요
      → DataHub에서 "PII" 태그로 검색
      → 23개 테이블에 개인정보 존재 확인
      → 접근 권한 정책 적용
```

## DataHub의 장점

1. **확장 가능한 아키텍처**: 마이크로서비스 기반으로 대규모 조직에 적합
2. **풍부한 메타데이터 모델**: Entity-Aspect 모델로 유연한 확장 가능
3. **강력한 Lineage**: 컬럼 레벨까지 추적 가능
4. **활발한 커뮤니티**: LinkedIn의 지속적인 개발 및 오픈소스 기여
5. **다양한 통합**: 50+ 데이터 소스 지원
6. **이벤트 기반 자동화**: Kafka 기반 실시간 메타데이터 업데이트

## DataHub의 도입 고려사항

**언제 도입해야 하는가?**

- 조직의 데이터 자산이 100개 이상일 때
- 데이터 팀이 5명 이상일 때
- 데이터 계보 추적이 필요할 때
- 데이터 거버넌스 정책이 필요할 때
- 데이터 검색에 많은 시간이 소요될 때

**도입 시 고려사항**

- 인프라 리소스: MySQL, Elasticsearch, Kafka 필요
- 러닝 커브: 메타데이터 모델 이해 필요
- 초기 설정: Ingestion 파이프라인 구성에 시간 소요
- 운영 부담: 메타데이터 품질 관리 필요
